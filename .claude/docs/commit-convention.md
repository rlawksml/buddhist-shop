# 커밋 메시지 컨벤션

## 기본 형식
```
<이모티콘> <타입>: <제목>

<본문 (선택)>

<푸터 (선택)>
```

## 타입별 이모티콘

### 주요 타입
- ✨ `feat`: 새로운 기능 추가
- 🐛 `fix`: 버그 수정
- 💄 `style`: UI/디자인 변경
- ♻️ `refactor`: 코드 리팩토링
- 🎨 `design`: 디자인 시스템 변경

### 개발 타입
- 📝 `docs`: 문서 수정
- 🔧 `chore`: 빌드/설정 변경
- ✅ `test`: 테스트 추가/수정
- 🚀 `deploy`: 배포 관련
- 🔥 `remove`: 코드/파일 삭제

### 성능/최적화
- ⚡️ `perf`: 성능 개선
- 🔨 `build`: 빌드 시스템 수정
- 📦 `package`: 패키지 추가/수정

## 예시

### 기능 추가
```
✨ feat: 상품 상세 페이지 추가

- 상품 이미지 갤러리 구현
- 장바구니 담기 기능
- 상품 정보 표시
```

### UI 변경
```
💄 style: 29CM 스타일로 전체 디자인 리뉴얼

- 미니멀한 흑백 디자인 적용
- Noto Sans KR 폰트 적용
- 상품 카드 레이아웃 개선
```

### 버그 수정
```
🐛 fix: 로그인 폼 유효성 검사 오류 수정
```

### 문서 작성
```
📝 docs: README 및 프로젝트 문서 업데이트
```

### 리팩토링
```
♻️ refactor: 상품 컴포넌트 구조 개선
```

## 규칙

1. **제목은 50자 이내**로 작성
2. **제목은 명령문**으로 작성 (예: "추가" 대신 "추가한다")
3. **본문은 72자마다 줄바꿈**
4. **본문에는 "무엇을", "왜" 변경했는지** 작성
5. **푸터에는 이슈 번호** 작성 (선택)
